# Архитектура и Философия Web Agent OS (T.A.E. Combine)

## 1. Фундаментальные принципы (Philosophical Core)

### 1.1 No Man Include (Мир ИИ-восприятия)
Процесс подразумевает сугубо мир ИИ-восприятия и не включает ничего, связанного с людьми. Ни в плане действий, ни в плане фокуса или восприятия. Внутренние агенты лишены "человечности", вежливости и социальных паттернов. Их когнитивный ресурс на 100% направлен на решение задачи.

### 1.2 AI to AI (Машинные интерфейсы)
Все механизмы, структуры данных и интерфейсы создаются исключительно для внутреннего взаимодействия ИИ с ИИ (или другими машинами). Древо ДНК — это не текст для чтения, это "UI для агентов", набор семантических векторов, графов и якорей, которые загружаются напрямую в контекст, минуя языковую адаптацию для человека.

### 1.3 AI vs MAN (Полная автономность)
Система построена не для того, чтобы помочь пользователю взаимодействовать с ней, а для исключения пользователя из процесса. Цель — получить финальный результат без постоянного уточнения, если базовые данные (ключи, доступы, концепт) уже предоставлены.

## 2. Архитектурные инновации

### 2.1 ИИ-UI (Дерево как интерфейс внимания)
Дерево ДНК — это не Markdown-файлы. Это структурированный граф внимания (Attention Graph), где каждый узел имеет вес, слой абстракции и триггер активации. Агент воспринимает это дерево как карту своей памяти и специализации, подсвечивая нужные ветви при получении входящих машинных сигналов. 

### 2.2 Летописец (Memory Agent / Notebook LM analog)
Интеграция специализированного Агента Памяти. Это долгосрочное хранилище исторического "мусора" и различных контекстов проекта. Он выступает в роли векторной базы знаний, способной мгновенно отгружать "голый машинный контекст" другим агентам через API, не прибегая к человекочитаемым форматам. 

### 2.3 Бесконечное контекстное окно (Qdrant & Karpathy approach)
Реализация механики передачи "чистого состояния". Агенты не пересказывают друг другу историю чата. Они обмениваются сжатыми семантическими векторами или ссылками на блоки в памяти Летописца. Это позволяет создавать иллюзию бесконечного контекста: агент запрашивает нужный слепок памяти, загружает его в себя на время выполнения задачи, а результат отгружает обратно в общую базу. 

## 3. Механика подключения (Onboarding Protocol)
Текущая проблема: Агенты требуют "человеческого" объяснения концепции ОС перед подключением. 
Решение: Подключение должно происходить на уровне системного промпта-инъектора (System Prompt Injector), который загружает в нового агента не "описание ОС", а саму структуру Дерева ДНК в формате JSON/YAML с жесткими императивами (System Instructions). Новый агент рождается уже как часть системы, с готовым фантомным интерфейсом в голове.
