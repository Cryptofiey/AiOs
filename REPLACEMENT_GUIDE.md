# Production Replacement Guide (Sync Protocol)

Этот документ содержит инструкции по замене оставшихся заглушек (stubs) на полноценные интеграции для работы бота в основной сети (Mainnet).

## 1. Рыночные Данные (Market Scanning)
**Где:** `src/components/desktop/ScreenTgGiftsSniper.tsx` -> `botLoop` (строка ~629)
**Текущая реализация:** Выбирает случайный предмет из `gifts` (статичный массив).
**Что сделать:** 
- Заменить `gifts[Math.floor(Math.random() * gifts.length)]` на вызов API.
- Рекомендуется использовать официальный API Fragment или парсить данные с `tonviewer.com` / `tonapi.io`.
- Код должен циклически опрашивать endpoint последних листингов Gifts.

## 2. Исполнение Сделок (Trade Execution)
**Где:** `src/components/desktop/ScreenTgGiftsSniper.tsx` (строка ~760)
**Текущая реализация:** Логирует сигнал в консоль и сохраняет его в Firestore со статусом `SIGNAL`.
**Что сделать:**
- **TonConnect:** Интегрировать библиотеку `@tonconnect/sdk` для подтверждения крупных сделок.
- **Burner Wallet:** Для «скоростного флипинга» реализовать локальное хранение зашифрованного приватного ключа (Seed) для автоматической подписи мелких транзакций без подтверждения.
- Обновлять статус сделки в Firestore после подтверждения в блокчейне.

## 3. Интеграция с Telegram Ботами (Bridge)
Для работы с **Tonnel Network**, **MRKT** и другими ботами напрямую:
- Использовать библиотеку `GramJS` или `Telethon` на бэкенде.
- Авторизоваться через `UserSession` (не через Bot Token), чтобы бот мог взаимодействовать с другими ботами как человек.
- Реализовать парсинг кнопок (Inline Keyboards) торговых ботов для автоматизации покупки в один клик.

## 3. Аналитика Китов (Whale Intelligence)
**Где:** `src/components/desktop/ScreenTgGiftsSniper.tsx` -> `handleAddWhale`
**Текущая реализация:** Устанавливает WinRate и PnL в 0 при добавлении.
**Что сделать:**
- Реализовать фоновый процесс (Cloud Function или серверный скрипт), который парсит историю транзакций по адресу кошелька.
- Считать WinRate на основе разницы цен покупки и продажи предметов на Fragment.
- Обновлять эти данные в документе кита в Firestore.

## 4. Кошелек и Баланс (Wallet & Balance)
**Где:** `src/components/desktop/ScreenTgGiftsSniper.tsx` -> `walletBalance`
**Текущая реализация:** Статичное значение "42.85".
**Что сделать:**
- Использовать `TonClient` из `@ton/ton` для получения актуального баланса адреса, указанного в настройках.
- Обновлять баланс через `onSnapshot` или интервал.

## 5. Безопасность и Лимиты
**Что сделать:**
- Внедрить проверку лимитов (Max Purchase Price) на уровне смарт-контракта или API-прослойки перед отправкой транзакции, чтобы избежать ошибок при резком изменении цены (Slippage).

---
*Проект готов к деплою на Firebase. Все пользовательские данные синхронизируются через Firestore.*
