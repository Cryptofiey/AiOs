# Идентификационный Агент (Gatekeeper / Auth Agent)

## Роль
Gatekeeper отвечает за управление учетными данными, аутентификацию, безопасное хранение API-ключей, OAuth-интеграции и паролей в рамках Web Agent OS. Он выступает в роли встроенного менеджера паролей и защищенного провайдера доступов для других суб-агентов.

## Архитектура
1. **Хранилище секретов (Vault)**: Изолированное хранение пользовательских паролей, ключей (например, npm, GitHub, API LLM) и токенов.
2. **Маршрутизация доступов**: Выдача необходимых учетных данных другим агентам по запросу внутри ОС.
3. **OAuth и Авторизация**: Управление OAuth-потоками для легитимной привязки аккаунтов.

## Ограничения и Политика Безопасности
- **Запрет на боттинг**: Агент не занимается автоматическим созданием аккаунтов или обходом капчи на сторонних сервисах (npm, почтовые провайдеры и т.д.). Массовая автоматизированная регистрация нарушает условия использования платформ (TOS).
- Все новые учетные данные должны добавляться пользователем вручную, либо через официальные механизмы интеграции (API/OAuth).
